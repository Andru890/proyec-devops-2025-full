#versions.tf:
Define la versión mínima requerida de Terraform y los proveedores necesarios para el proyecto.
Se asegura de que se utilicen las versiones correctas de Terraform y los proveedores.




provider.tf:
Configura el proveedor de AWS, incluyendo las credenciales y los endpoints para LocalStack.
Define cómo Terraform interactuará con los servicios de AWS simulados por LocalStack.



variables.tf:
Define las variables que se utilizarán en el proyecto.
Permite parametrizar la configuración de Terraform, facilitando la reutilización y la gestión de diferentes entornos (como dev y prod).


main.tf:
Es el archivo principal de configuración de Terraform.
Define los módulos que se utilizarán para crear los recursos de infraestructura, como VPC, RDS y EC2.
Los módulos permiten organizar y reutilizar configuraciones de Terraform.



outputs.tf:
Define las salidas de Terraform, que son valores que se pueden utilizar después de que la infraestructura ha sido creada.
Proporciona información sobre los recursos creados, como el ID de la VPC, el endpoint de RDS y la IP pública de la instancia EC2.


terraform.tfvars en environments/dev:
Contiene los valores específicos para el entorno de desarrollo (dev).
Define las variables que se utilizarán en este entorno, como el CIDR de la VPC, el usuario y la contraseña de la base de datos.


------------------------------------------------------------------------------------------------------------------------------------------------
MODULOS

modules/vpc:
Contiene los archivos main.tf y variables.tf para definir y parametrizar la creación de la VPC.
main.tf define el recurso de la VPC y variables.tf define las variables necesarias para el módulo.




modules/rds:
Contiene los archivos main.tf y variables.tf para definir y parametrizar la creación de la instancia RDS.
main.tf define el recurso de la instancia RDS y variables.tf define las variables necesarias para el módulo.




modules/ec2:
Contiene los archivos main.tf y variables.tf para definir y parametrizar la creación de la instancia EC2.
main.tf define el recurso de la instancia EC2 y variables.tf define las variables necesarias para el módulo.







Resumen de la relación
versions.tf y provider.tf configuran el entorno de Terraform y el proveedor de AWS.
variables.tf define las variables que se utilizarán en el proyecto.
main.tf utiliza estas variables y módulos para definir los recursos de infraestructura.
outputs.tf define las salidas que se pueden utilizar después de la creación de la infraestructura.
terraform.tfvars en environments/dev proporciona valores específicos para el entorno de desarrollo, permitiendo la parametrización y reutilización de la configuración de Terraform.
Los módulos en modules/vpc, modules/rds y modules/ec2 contienen la lógica específica para crear los recursos de infraestructura y se referencian desde main.tf.
